# 2048 — Complete Game Specification

## 1. Game Overview

**Title:** 2048
**Genre:** Sliding tile puzzle
**Platform:** Single-player, keyboard/swipe-controlled
**Original Release:** March 2014 (Gabriele Cirulli)
**Objective:** Slide numbered tiles on a 4x4 grid. When two tiles with the same number collide, they merge into one tile with their sum. Reach the 2048 tile to win; continue playing for a higher score. The game ends when no moves are possible.

## 4. Core Mechanics

### 4.1 Tile Sliding

When the player inputs a direction (Up, Down, Left, Right):

1. All tiles slide as far as possible in that direction.
2. If two tiles with the same value collide during the slide, they merge into a single tile with double the value.
3. A tile can only merge once per move (a merged tile cannot merge again in the same move).
4. Merges are processed in the direction of movement (e.g., sliding right processes from the rightmost column first).

### 4.2 Merge Rules (Detailed)

For a rightward slide of a row [2, 2, 2, 2]:
- Process from right: Rightmost pair (positions 2,3) merge to 4 at position 3.
- Next pair (positions 0,1) merge to 4 at position 2.
- Result: [_, _, 4, 4]

For a rightward slide of a row [2, 2, 4, _]:
- Position 2 (value 4) slides to position 3: [2, 2, _, 4]
- Positions 0,1 (both value 2) merge: [_, _, 2+2=4, 4]
- Result: [_, _, 4, 4]

For a rightward slide of a row [2, 4, 2, 2]:
- Positions 2,3 (both value 2) merge to 4 at position 3.
- Position 1 (value 4) slides to position 2.
- Position 0 (value 2) slides to position 1.
- Result: [_, 2, 4, 4]

### 4.3 Slide Algorithm (Per Row/Column)

```python
def slide_line(line, direction='right'):
    # Remove zeros (empty cells)
    tiles = [t for t in line if t != 0]

    if direction in ('right', 'down'):
        tiles = tiles[::-1]  # reverse for right/down processing

    merged = []
    skip = False
    for i in range(len(tiles)):
        if skip:
            skip = False
            continue
        if i + 1 < len(tiles) and tiles[i] == tiles[i + 1]:
            merged.append(tiles[i] * 2)
            skip = True  # next tile consumed
        else:
            merged.append(tiles[i])

    # Pad with zeros
    while len(merged) < 4:
        merged.append(0)

    if direction in ('right', 'down'):
        merged = merged[::-1]

    return merged
```

### 4.4 Move Validity

A move is valid if and only if at least one tile moves to a different position or at least one merge occurs. If a move would result in no change to the board, the move is rejected (no new tile spawns).

## 5. Tile Spawning

### 5.1 Spawn Rules

After every valid move:
1. Find all empty cells on the grid.
2. Select one empty cell uniformly at random.
3. Place a new tile with value 2 (90% probability) or 4 (10% probability).

### 5.2 Initial Setup

At game start:
1. Board is completely empty (all 16 cells = 0).
2. Spawn two tiles using the standard spawn rules (one at a time, each in a random empty cell).

## 6. Scoring

### 6.1 Score Calculation

- Score starts at 0.
- When two tiles merge, the value of the resulting tile is added to the score.
- Example: Two 16-tiles merge into a 32-tile. Score += 32.
- Multiple merges in a single move are all added.
- Score is cumulative across the entire game.

### 6.2 Best Score

- Track the highest score ever achieved across all games.
- Stored persistently.

### 6.3 Theoretical Maximum

- The theoretical maximum tile is 131072 (2^17), though practically unreachable.
- Maximum possible score is approximately 3,932,100.

## 7. Win and Game Over

### 7.1 Win Condition

- The player wins when any tile reaches the value 2048.
- Upon winning, display a "You Win!" overlay.
- Offer "Keep Playing" to continue beyond 2048.
- The game tracks whether the player has already won (don't show the win screen again if continuing).

### 7.2 Game Over Condition

The game is over when:
1. All 16 cells are occupied (no empty cells), AND
2. No two adjacent tiles (horizontally or vertically) have the same value.

```python
def is_game_over(board):
    for r in range(4):
        for c in range(4):
            if board[r][c] == 0:
                return False
            if c + 1 < 4 and board[r][c] == board[r][c+1]:
                return False
            if r + 1 < 4 and board[r][c] == board[r+1][c]:
                return False
    return True
```

Swipe detection:
- Minimum swipe distance: 30 pixels.
- Determine direction by comparing deltaX and deltaY:
  - If |deltaX| > |deltaY|: horizontal swipe (left if deltaX < 0, right if deltaX > 0).
  - If |deltaY| > |deltaX|: vertical swipe (up if deltaY < 0, down if deltaY > 0).

### 8.3 Input Debouncing

- Ignore inputs during tile animation (150ms).
- Queue at most 1 pending input during animation.

## 11. Undo System (Optional Feature)

- Store the previous board state and score.
- "Undo" button reverts to the last state.
- Only one level of undo (cannot undo twice in a row).
- After undo, the undo button is disabled until the next move.
- Undo removes the tile that was spawned and reverses all merges.

## 16. State Machine

```
[INITIAL]
    --> [NEW_GAME]
        --> Spawn 2 tiles
        --> [WAITING_FOR_INPUT]

[WAITING_FOR_INPUT]
    --> (direction input)
        --> [CHECK_VALID_MOVE]
            --> (valid)
                --> [ANIMATE_SLIDE] (150ms)
                --> [SPAWN_NEW_TILE] (200ms appear animation)
                --> [CHECK_WIN]
                    --> (2048 reached, first time)
                        --> [WIN_SCREEN]
                            --> (keep going) --> [WAITING_FOR_INPUT]
                            --> (try again) --> [NEW_GAME]
                    --> (no win)
                        --> [CHECK_GAME_OVER]
                            --> (game over) --> [GAME_OVER_SCREEN]
                                --> (try again) --> [NEW_GAME]
                            --> (not over) --> [WAITING_FOR_INPUT]
            --> (invalid, no change)
                --> [WAITING_FOR_INPUT]
    --> (new game button)
        --> [CONFIRM_NEW_GAME] (if game in progress)
            --> [NEW_GAME]
    --> (undo button)
        --> [UNDO] --> [WAITING_FOR_INPUT]
```

## 19. Edge Cases

1. **Full board, one merge possible:** The game is NOT over. The merge can free a cell.
2. **Multiple merges in one row:** Each pair merges independently. [2,2,2,2] sliding right becomes [_,_,4,4], NOT [_,_,_,8].
3. **Zero-value tiles:** Represent empty cells. Never displayed.
4. **Score counter animation:** If multiple merges happen simultaneously, the score popup shows the total for that move.
5. **Tile value overflow:** Theoretically impossible with standard int types, but cap at 131072 if needed.
6. **Rapid input:** If the player inputs a direction while animations are still playing, queue the input and execute after animations complete. Queue size: 1 (only the most recent queued input is kept).
7. **New game during active game:** Show confirmation dialog: "Start a new game? Current progress will be lost."
